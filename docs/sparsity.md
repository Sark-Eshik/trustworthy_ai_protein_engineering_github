# Sparsity Framework Documentation

This document covers the complete structure, mathematics, implementation, and certification procedures for the Sparsity submodules within the Trustworthy AI Protein Engineering framework.

---

## 1. Empirical Sparsity Submodule

### Overview
The **Empirical Sparsity Submodule** measures experimental rarity of single amino acid mutations based on their occurrences in large-scale experimental assays (such as Megascale-D mutational datasets). 

A mutation observed frequently is assumed to represent a heavily explored, dense region of sequence space. Conversely, a mutation observed rarely or not at all is situated in a sparse region of sequence space, indicating a potential lack of physical or thermodynamic characterization.

### Core Science
For each mutation $i$, empirical sparsity is calculated as:

$$\text{Frequency}_i = \frac{\text{Count}_i}{\sum_{j} \text{Count}_j}$$

$$\text{Normalized Frequency}_i = \frac{\text{Frequency}_i}{\max(\text{Frequency})}$$

$$\text{Empirical Sparsity}_i = 1.0 - \text{Normalized Frequency}_i$$

where:
- $\text{Empirical Sparsity} = 0.0$ represents the most frequently observed (densest) mutation.
- $\text{Empirical Sparsity} = 1.0$ represents the rarest/unobserved mutation.

### Directory Structure
```
src/sparsity/empirical/
├── __init__.py           # Submodule exports
├── __main__.py           # Package-level main entry point
├── empirical_sparsity.py # Main calculation engine
└── empirical_validation.py # Scientific and schema validator
```

### CLI Execution
To compute empirical sparsity from a counts CSV:
```bash
PYTHONPATH=. python -m src.sparsity.empirical \
  --input data/raw/empirical/mutation_counts.csv \
  --output results/empirical/
```

To validate the generated output file scientifically and structurally:
```bash
PYTHONPATH=. python -m src.sparsity.empirical.empirical_validation \
  --file results/empirical/empirical_sparsity.parquet
```

### Dataset Specifications (D101)
The generated parquet dataset contains the following schema:
- `mutation_id` (Primary Key, String): Unique identifier for the single point mutation.
- `count` (Int64): Number of raw experimental observations.
- `frequency` (Float64): Proportion of total observations represented by this mutation.
- `mutation_frequency` (Float64): Alias for `frequency`.
- `normalized_frequency` (Float64): Frequency scaled by the maximum observed frequency in the dataset.
- `empirical_sparsity` (Float64): Calculated empirical sparsity in the range $[0.0, 1.0]$.

### Validation Checks
The validator certifies that:
1. Schema matches the official D101 definition.
2. Sparsity values reside strictly within the $[0.0, 1.0]$ bounds.
3. No duplicate mutation keys or null values exist.
4. Correct scientific ranking (mutations with smaller observation counts must have higher sparsity scores).
5. At least one mutation (the densest) achieves exactly $0.0$ sparsity.

## 2. Evolutionary Sparsity Submodule

### Overview
The **Evolutionary Sparsity Submodule** measures evolutionary rarity of mutations using ESM2 (Evolutionary Scale Modeling) likelihood scoring. 

A mutation that achieves high conditional probability under the ESM2 masked language model is presumed to be evolutionarily plausible, while a mutation with extremely low probability is evolutionarily unusual and receives higher evolutionary sparsity.

### Core Science
For each mutation $i$:

$$\text{Log Probability}_i = -\log(P_i)$$

where $P_i$ is the conditional probability of the mutant residue at position $x$ generated by ESM2 (by masking position $x$ and extracting logits over standard amino acids).

Sparsity is then normalized using min-max scaling of the log-likelihood values:

$$\text{Evolutionary Sparsity}_i = \frac{\text{Log Probability}_i - \min_j(\text{Log Probability}_j)}{\max_j(\text{Log Probability}_j) - \min_j(\text{Log Probability}_j)}$$

where:
- $\text{Evolutionary Sparsity} = 0.0$ represents the most plausible (least surprising) mutation.
- $\text{Evolutionary Sparsity} = 1.0$ represents the most unusual (highest surprise) mutation.

### Validation Checks

3. No duplicate mutation keys or null values exist.

5. Minimum probability mutation achieves exactly $1.0$ sparsity, and maximum probability mutation achieves exactly $0.0$ sparsity.

## 3. Structural Sparsity Submodule

### Overview
The **Structural Sparsity Submodule** measures local residue environment constraints using Solvent Accessible Surface Area (SASA). 

A residue situated deep inside the protein core has low solvent exposure (low SASA) due to steric compaction and burial, receiving higher structural sparsity. In contrast, an exposed, surface-accessible residue has high SASA and receives low structural sparsity.

### Core Science
For each residue coordinate $x$:

$$\text{Normalized SASA}_x = \frac{\text{SASA}_x}{\max_y(\text{SASA}_y)}$$

$$\text{Structural Sparsity}_x = 1.0 - \text{Normalized SASA}_x$$

where:
- $\text{Structural Sparsity} = 0.0$ represents the most solvent-exposed (surface) residue.
- $\text{Structural Sparsity} = 1.0$ represents the most buried (protein core) residue.

```

The generated parquet dataset contains the following schema:

- `sasa` (Float64): Calculated Solvent Accessible Surface Area in $\text{\AA}^2$.
- `normalized_sasa` (Float64): SASA scaled by the maximum observed SASA in the structure.
- `structural_sparsity` (Float64): Normalized structural sparsity in the range $[0.0, 1.0]$.

### Validation Checks
The validator certifies that:
1. Schema matches the official D103 definition.
2. Value bounds are valid (SASA is positive, normalized SASA and sparsities reside strictly in $[0.0, 1.0]$).
3. No duplicate mutation keys or null values exist.
4. Correct scientific ranking (mutations on more buried positions must have higher structural sparsity scores).
5. Maximum SASA position achieves exactly $0.0$ structural sparsity.

## 4. Combined Sparsity Submodule

### Overview
The **Combined Sparsity Submodule** mathematically integrates multiple distinct sparsity perspectives (experimental exploration density, evolutionary likeliness, and 3D residue burial constraints) into a single, unified mathematical index. This dataset acts as the primary explanatory variable throughout the downstream thermodynamics consistency (Experiment 1) and epistasis predictability (Experiment 2) analyses.

### Core Science
To accommodate both high-throughput exploration contexts (Megascale-D datasets) and applied design contexts where empirical frequencies are absent (the Transketolase - TKT industrial carbon-capture pipeline), the combined framework supports two operational modes:

#### 1. Megascale-D Mode
Used when experimental, evolutionary, and structural datasets are all available:

$$\text{Combined Sparsity}_i = \frac{\text{Empirical Sparsity}_i + \text{Evolutionary Sparsity}_i + \text{Structural Sparsity}_i}{3.0}$$

#### 2. TKT Industrial Mode
Used when empirical frequencies are absent:

$$\text{Combined Sparsity}_i = \frac{\text{Evolutionary Sparsity}_i + \text{Structural Sparsity}_i}{2.0}$$

### Interpretation Range
The resulting score represents sequence space isolation:
- **0.00 – 0.25**: Very Dense (highly characterized, evolutionarily common, fully surface exposed)
- **0.25 – 0.50**: Moderately Dense
- **0.50 – 0.75**: Sparse
- **0.75 – 1.00**: Highly Sparse (completely uncharacterized, evolutionarily unexpected, buried deep in protein core)

### Directory Structure
```
src/sparsity/combined/
├── __init__.py               # Submodule exports
├── __main__.py               # Package-level main entry point
├── combine_sparsity.py       # Pipeline orchestrator & mode arithmetic solver
└── combined_validation.py    # Scientific math-audit and schema validator
```

### CLI Execution
To compute combined sparsity (Megascale-D Mode):

```

To compute combined sparsity (TKT Industrial Mode):
```bash
PYTHONPATH=. python -m src.sparsity.combined \
  --mode tkt \
  --evolutionary results/evolutionary/evolutionary_sparsity.parquet \
  --structural results/structural/structural_sparsity.parquet \
  --output results/combined/
```

To validate the generated output file scientifically and structurally:
```bash
PYTHONPATH=. python -m src.sparsity.combined.combined_validation \
  --file results/combined/combined_sparsity.parquet \
  --mode megascale_d
```

### Dataset Specifications (D104)
The generated parquet dataset contains the following schema:
- `mutation_id` (Primary Key, String): Unique identifier for the single point mutation.
- `empirical_sparsity` (Float64, Nullable): Experimental rarity score (permitted to hold nulls in TKT Mode).
- `evolutionary_sparsity` (Float64): Likelihood-based rarity score under ESM2.
- `structural_sparsity` (Float64): Burial-based rarity score under FreeSASA.
- `combined_sparsity` (Float64): Multi-dimensional unified sparsity average in the range $[0.0, 1.0]$.

### Validation Checks
The validator certifies that:
1. Schema matches the official D104 definition (accounting for allowed nullable empirical_sparsity in TKT Mode).
2. Combined sparsity scores reside strictly within $[0.0, 1.0]$.
3. No duplicate mutation keys exist.
4. Combined values pass an **Arithmetic Recomputation Audit** to ensure $Combined = Average(Constituents)$ is mathematically perfect.
